인터넷이 동작하는 기본적인 원리를 알아봅시다.

인터넷 동작의 본질적인 필요 매개체: 2대의 컴퓨터
한대는 Web 서버 다른 한대는 web 브라우저. 두대는 연결된 상태
서버는 HTML파일이 들어있는 제공자. 클라이언트는 그것을 요청하는 사용자.
클라이언트의 요청: request || 서버의 응답: respone
웹서버의 설치와 웹서버의 호스팅. 2가지 갈림길이 있음.
호스팅을 먼저 공부한 후 서버 설치에 대해 배웁시다.

직접 웹서버를 운영하는 것은 쉽지 않은 일.
인터넷에 연결된 컴퓨터 하나하나를 Host라고 함.
이런 HOST를 제공하는 회사를 웹호스트 업체라 함
수 많은 웹호스팅 업체가 존재. 이 중 무료+유명한 서비스인 Github의 pages 기능 이용

깃헙 pages 기능 이용법

  1. 회원 가입(sing up) 및 로그인(sign in) 하기.
  2. 저장소(Repositories) 생성.
  3. 이름은 web1 공개/비공개는 개인 취향
  4. 저장소에서 +버튼을 클릭후 Upload files 선택
  5. Drag앤Drop 혹은 직접 파일 선택한
  6. 하단 창에서 변경된 내용을 입력
  7. 초록색버튼(Commit chages) 클릭하여 업로드

업로드한 웹 페이지 서비스 하기

  1. web1 저장소에서 setting 설정으로 이동
  2. Pages탭 선택
  3. Branch를 main 혹은 master를 선택하고 Save 버튼을 누릅니다.
  4. 상단에 사용자 마다 다른 주소가 표시 (뜨지 않으면 5분 정도후 재접속)
  5. 기본 접속은 Readme.md 파일
  6. 다른 파일 접속은 https://mowja.github.io/web1/Hello.html 같이 마지막에 파일명.html을 입력

이것으로 나만의 페이지를 인터넷에 호스팅까지 완성!
